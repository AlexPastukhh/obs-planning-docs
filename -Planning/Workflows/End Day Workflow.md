# End Day Workflow

Status: active workflow.

Purpose: close a date-based day file and write final day summary.

## Uses

- `Workflows/Use Day File Workflow.md`
- `Workflows/Support Score Review Workflow.md`
- `Days/YYYY/YYYY-MM-DD.md`
- `Support Score Guide.md`
- `Templates/Support Score Review Template.md`
- `Templates/Planning State Output Template.md`
- `Templates/Default Dashboard Template.md`

## Steps

1. Identify date and day file.
2. Summarize Work Score from Finished Sessions.
3. Explain goal outcome using Desired / Undesired and Progress Signal.
4. Summarize main course movement.
5. Note CV / SD / EX only if relevant.
6. If support review is requested or appropriate at close / next morning, run `Support Score Review Workflow.md`.
7. Fill `Final Day Summary`.
8. Set day file `Status: closed` if the user is closing the day.
9. Render Mnemonic Emoji Table.
10. Render final compact Planning State Output.
11. Render Support Review if calculated or requested.

Detailed day summary lives in the date-based day file.

Do not write detailed day summary to `Session Log.md` by default.

Do not render full Dashboard Core by default.
