## Meaning

- 🎯Рез = Real Fun vs Stimulation + Real Pain vs Chemistry + predicted Desired/Undesired + готов получить сейчас + не принимаю слив + ради Result Tracking.
- Real Fun vs Stimulation = отличить реальное удовольствие / облегчение / контроль / улучшение жизни от стимуляции.
- Стимуляция соперничает с реально хорошими вещами: она может казаться удовольствием, безопасностью или облегчением, но забирать ресурс и шанс получить то, что реально улучшит состояние.
- Desired строится как аналитическое предсказание реального удовольствия / контроля / облегчения, а не как реакция на текущую стимуляцию.
- Real Pain vs Chemistry = отличить реальную плохую цену / слив / ухудшение положения от временного дискомфорта, стартового сопротивления и химикатов.
- Undesired строится как аналитическое предсказание реальной плохой цены / слива, а не как реакция на текущий дискомфорт.
- 🧵📜>🧪Обещ = Chem vs Promise + Fire / Wood.
- Chem vs Promise = химикаты не ты -> обещание сильнее химии -> эффект может появиться после действия -> пришить обещание как источник истины.
- Fire / Wood = не разжигать химикаты/стимулы -> не подкидывать лишнему -> химикаты дать догореть -> стимулы переключить на нужные -> дрова не бесконечные.
- Не разжигать химикаты/стимулы = не провоцировать то, что начнёт сильнее тянуть, шуметь, обещать стимуляцию и уносить от Desired.
- 📊Скор = как сделать сессию хорошей по D/F/K/P, темпу, guard и boost.
- 🧲ЗИ = занос / инерция / привычка / будущие действия.
- 🧲⚡ = stimulus pull / easy stimulation / fast attention magnet. Use this instead of `🎪` for stimulus meaning.
- 🚂🛤️ = rails / process on rails. Bare `🛤️` means course.

## Rule

Default Dashboard is the compact mnemonic/action core only.

In planning/session-control mode, Mnemonic Emoji Table may be shown by default, but full Dashboard Core is rendered only on explicit dashboard request.

Mnemonic Emoji Table is mostly emoji/symbol based by design. Short group headers are allowed when they improve memory and orientation.

The table can be omitted only for narrow technical answers, diff/file checks, archive/package tasks, or when the user explicitly asks for a short answer.

It may be shown with Planning State Output, but it does not replace Planning State Output.
